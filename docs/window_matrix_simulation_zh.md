# 窗戶時段、天氣、季節矩陣模擬

本文件說明新增的窗戶模擬矩陣。目的不是建立完整建築物理模型，而是讓論文第五章可以展示「窗戶作為環境變數」在不同外部條件下，對單房間三個環境因素造成的可解釋影響。

## 模擬範圍

目前固定產生 48 組窗戶情境：

- 時段：早上、中午、下午、晚上
- 天氣：陰天、晴天、雨天
- 季節：春季、夏季、秋季、冬季

情境命名格式如下：

```text
window_<season>_<weather>_<time>
```

例如：

```text
window_summer_sunny_noon
window_winter_rainy_night
```

## 模型設定

每個窗戶矩陣情境都採用相同房間幾何與 8 顆角落感測器設定，並只啟用 `window_main`：

- `ac_main`：關閉
- `window_main`：70% 等效開啟程度
- `light_main`：關閉

目標區域改為 `window_zone`，用來觀察靠窗區域最直接的三因子變化；同時輸出 `center_zone` 與 `door_side_zone`，方便比較窗戶影響如何隨距離衰減。

## 外部條件參數

季節決定室內基準與外部基準：

| 季節 | 室內基準溫度 | 室內基準濕度 | 外部基準溫度 | 外部基準濕度 | 基準日照 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 春季 | 24.0 | 63.0 | 23.0 | 68.0 | 26000 |
| 夏季 | 29.0 | 70.0 | 33.0 | 76.0 | 36000 |
| 秋季 | 25.0 | 62.0 | 26.0 | 64.0 | 28000 |
| 冬季 | 19.0 | 58.0 | 16.0 | 66.0 | 19000 |

天氣修正外部溫度、濕度與日照係數：

| 天氣 | 溫度修正 | 濕度修正 | 日照係數 |
| --- | ---: | ---: | ---: |
| 陰天 | 0.0 | +2.0 | 0.35 |
| 晴天 | +2.0 | -5.0 | 1.00 |
| 雨天 | -2.0 | +12.0 | 0.08 |

時段修正外部溫度與日照係數：

| 時段 | 溫度修正 | 日照係數 |
| --- | ---: | ---: |
| 早上 | -1.5 | 0.55 |
| 中午 | +2.0 | 1.00 |
| 下午 | +1.0 | 0.72 |
| 晚上 | -3.0 | 0.01 |

## 執行方式

產生 JSON 結果：

```bash
python3 scripts/run_window_matrix.py
```

輸出檔案：

```text
outputs/window_matrix_summary.json
```

也可以透過 MCP tool 執行：

```text
run_window_matrix
```

或在 web demo 中查看 `Window Season/Weather/Time Matrix` 表格。

## 初步結果範例

目前產出的例子：

| 情境 | 外部溫度 | 外部濕度 | 外部日照 | 窗戶區溫度 | 窗戶區濕度 | 窗戶區照度 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| `window_summer_sunny_noon` | 37.0 | 71.0 | 36000.0 | 29.4703 | 70.4423 | 223.9044 |
| `window_winter_rainy_night` | 11.0 | 78.0 | 15.2 | 18.2008 | 59.5622 | 68.8988 |
| `window_spring_cloudy_morning` | 21.5 | 70.0 | 5005.0 | 23.6372 | 63.7959 | 90.3925 |

這些結果符合目前模型假設：晴天中午引入較高照度與較高外部熱交換，雨天晚上則幾乎不提供日照，但會因外部濕度較高而使靠窗區濕度上升。

## 論文使用方式

建議在第五章把這組矩陣作為「外部環境變數敏感度分析」：

1. 比較不同時段下，窗戶對照度場的影響。
2. 比較不同天氣下，窗戶對濕度場的影響。
3. 比較不同季節下，窗戶對溫度場的影響。
4. 比較 `window_zone` 與 `center_zone`，說明窗戶影響場的距離衰減。

這組模擬能支撐你的核心問題：即使窗戶不是連網裝置，系統仍可透過外部條件假設與室內感測器校正，估計它對溫度、濕度與照度分布造成的影響。
