# 研究計畫草案

## 論文題目

英文題目：

**A Sparse-Sensing Spatial Digital Twin for Learning Environmental Impacts of Non-Networked Appliances in a Single Room**

中文題目：

**單房間非連網家電環境影響學習之稀疏感測空間數位孿生原型**

## 研究背景

智慧建築與智慧居家系統通常需要掌握室內環境狀態，才能進一步支援舒適度評估、能源管理與設備控制。然而，實際場域中不可能在空間中每一個位置都安裝感測器，因此系統往往只能取得少量離散點位資料。若僅依賴單點或少數點位量測，容易忽略同一房間內不同區域的溫度、濕度與照度差異。

另一個常見限制是，許多既有家電或環境裝置並非智慧裝置，無法連網，也沒有 API 可直接回報自身狀態或運轉功率。例如傳統冷氣、手動窗戶、一般照明設備雖然會改變室內環境，卻不一定能被系統直接讀取。本研究因此將問題定義為：在裝置本身缺乏連網能力的情況下，能否透過有限環境感測資料學習裝置對空間中溫度、濕度與照度造成的影響，並進一步支援更準確的環境控制。

本研究以單一房間作為研究對象，建立一個簡化但可解釋的空間數位孿生模型。模型透過 8 顆角落感測器估計室內三個環境因素的空間分布，並用可學習與可校正的設備影響函數描述冷氣、窗戶與照明對室內不同區域的影響。為支援互動查詢與系統整合，本研究另提供本地服務介面，其中包含 MCP server 與 web demo，讓外部 AI client 或使用者可查詢情境、執行模擬與取得設備動作排序。

## 研究動機

一般室內環境監測系統多半只回答「目前某個感測器位置的數值是多少」。若裝置本身沒有連網，系統也無法直接知道某台家電是否啟動、啟動後影響範圍多大、或它對不同區域造成何種變化。本研究希望進一步回答四個更接近數位孿生與智慧控制的問題：

1. 如果只知道房間角落的感測值，是否能估計房間內其他位置的溫度、濕度與照度？
2. 若冷氣、窗戶、照明等裝置沒有連網，是否能從環境變化中學習其影響函數？
3. 如果新增或啟用某個裝置，是否能推估它對不同區域的影響？
4. 在學習裝置影響後，模型是否能更準確地控制或推薦控制三個環境變數？
5. 是否能將學習後的數位孿生模型封裝為標準化工具介面，使 AI agent 能存取模擬與推薦能力？

## 研究目的

本研究的主要目的如下：

1. 建立單房間三因子空間數位孿生模型。
2. 定義溫度、濕度與照度三個環境場的簡化動態表示。
3. 建立非連網冷氣、窗戶與照明的參數化影響函數。
4. 使用 8 顆角落感測器對空間場與裝置影響進行殘差校正。
5. 根據學習後的裝置影響，建立三個環境變數的候選控制動作排序方法。
6. 將模型能力封裝為本地服務介面，並以 MCP server 作為其中一種工具化存取方式。
7. 以模擬案例驗證模型的合理性、可解釋性與穩定性。

## 研究範圍

本研究限定於單一矩形房間，不處理多房間或跨空間空氣交換。感測器配置固定為天花板四角與地面四角，共 8 顆角落感測器。每一顆感測器皆假設可取得溫度、濕度與照度資料。

設備範圍包含三類：

- 冷氣：主要影響溫度，次要影響濕度。
- 窗戶：受外部溫濕度與日照條件影響，同時影響三個環境因素。
- 照明：主要影響照度，次要影響溫度。

本研究假設裝置本身可被手動或外部方式操作，但不假設裝置會主動回報自身狀態。控制功能僅做到設備候選動作排序，不做自動閉環控制。MCP 部分限定為本地 stdio MCP server，不處理遠端部署、OAuth 或多使用者權限管理。

## 研究方法

本研究將室內狀態定義為三個空間與時間函數：

```text
T(x, y, z, t): temperature field
H(x, y, z, t): humidity field
L(x, y, z, t): illuminance field
```

任一環境因素 `v` 的估計場可表示為：

```text
F_v(x, y, z, t) = B_v(x, y, z) + Σ I_j,v(x, y, z, t) + C_v(x, y, z)
```

其中 `B_v` 為背景場，`I_j,v` 為第 `j` 個設備的影響函數，`C_v` 為由感測器殘差推估出的校正場。對於非連網裝置，`I_j,v` 並非由裝置 API 直接提供，而是由裝置啟用前後的環境變化、感測器殘差與模型參數校正逐步估計。

本研究採用連續場概念建立模型，但在實作上將房間離散為 3D 採樣網格，以便進行數值計算、熱圖輸出與區域平均分析。

為支援 AI agent 存取，本研究將核心模型封裝為可工具化的本地服務介面，其中一種實作為 MCP tools：

- `list_scenarios`：列出內建模擬情境。
- `run_scenario`：執行指定情境並回傳重建誤差與目標區域估計。
- `rank_actions`：依舒適度改善分數排序候選設備動作。
- `sample_point`：估計指定座標的溫度、濕度與照度。

## 預期貢獻

本研究的預期貢獻包含：

1. 提出一個不依賴高成本 CFD 模擬的室內三因子空間數位孿生框架。
2. 建立可由少量角落感測器校正的空間場估計方法。
3. 將非連網冷氣、窗戶與照明轉換為可學習、可比較的設備影響函數。
4. 提供目標區域導向的三因子控制動作排序方法。
5. 提供可工具化的服務介面，使 AI client 能以標準化方式存取模型。
6. 產出可重現的 Python 原型與模擬案例，支援後續實體部署或論文延伸。

## 初步研究假設

1. 單一房間內的環境因素可用簡化連續場近似表示。
2. 冷氣、窗戶與照明的空間影響可用距離衰減與方向性權重近似。
3. 8 顆角落感測器足以支撐 active device power scale 校準與 8 參數 trilinear residual correction。
4. 溫度與照度是本研究的主要變數，濕度作為次核心變數保留。
5. 非連網裝置的影響可由裝置啟用前後的環境變化進行近似學習。
6. 模型輸出的設備排序應具備可解釋性，而不僅是數值最佳化結果。

## 預期成果

研究完成後應包含以下成果：

1. 數位孿生模型公式與系統架構。
2. Python 模擬原型。
3. 8 組設備組合情境模擬。
4. 溫度、濕度與照度空間熱圖。
5. 校正前後誤差分析。
6. 候選設備動作排序結果。
7. 本地服務介面與 AI client 串接範例，其中包含 stdio MCP server。
8. 模型限制與未來實體部署建議。
