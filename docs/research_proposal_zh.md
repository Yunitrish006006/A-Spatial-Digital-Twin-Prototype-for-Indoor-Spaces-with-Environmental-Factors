# 研究計畫草案

## 論文題目

英文題目：

**A Lightweight Spatial Digital Twin Prototype for Single-Room Temperature, Humidity, and Illuminance Estimation Using Corner Sensor Calibration**

中文題目：

**基於角落感測器校正之單房間溫度、濕度與照度輕量化空間數位孿生原型**

## 研究背景

智慧建築與智慧居家系統通常需要掌握室內環境狀態，才能進一步支援舒適度評估、能源管理與設備控制。然而，實際場域中不可能在空間中每一個位置都安裝感測器，因此系統往往只能取得少量離散點位資料。若僅依賴單點或少數點位量測，容易忽略同一房間內不同區域的溫度、濕度與照度差異。

本研究以單一房間作為研究對象，建立一個簡化但可解釋的空間數位孿生模型。模型透過 8 顆角落感測器估計室內三個環境因素的空間分布，並用可參數化的設備影響函數描述冷氣、窗戶與照明對室內不同區域的影響。

## 研究動機

一般室內環境監測系統多半只回答「目前某個感測器位置的數值是多少」。本研究希望進一步回答三個更接近數位孿生的問題：

1. 如果只知道房間角落的感測值，是否能估計房間內其他位置的溫度、濕度與照度？
2. 如果新增或啟用冷氣、窗戶、照明等設備，是否能推估它們對不同區域的影響？
3. 如果使用者指定目標區域，模型是否能根據環境狀態推薦較合適的設備動作？

## 研究目的

本研究的主要目的如下：

1. 建立單房間三因子空間數位孿生模型。
2. 定義溫度、濕度與照度三個環境場的簡化動態表示。
3. 建立冷氣、窗戶與照明的參數化影響函數。
4. 使用 8 顆角落感測器對空間場進行殘差校正。
5. 建立候選設備動作的舒適度改善排序方法。
6. 以模擬案例驗證模型的合理性、可解釋性與穩定性。

## 研究範圍

本研究限定於單一矩形房間，不處理多房間或跨空間空氣交換。感測器配置固定為天花板四角與地面四角，共 8 顆角落感測器。每一顆感測器皆假設可取得溫度、濕度與照度資料。

設備範圍包含三類：

- 冷氣：主要影響溫度，次要影響濕度。
- 窗戶：受外部溫濕度與日照條件影響，同時影響三個環境因素。
- 照明：主要影響照度，次要影響溫度。

控制功能僅做到設備候選動作排序，不做自動閉環控制。

## 研究方法

本研究將室內狀態定義為三個空間與時間函數：

```text
T(x, y, z, t): temperature field
H(x, y, z, t): humidity field
L(x, y, z, t): illuminance field
```

任一環境因素 `v` 的估計場可表示為：

```text
F_v(x, y, z, t) = B_v(x, y, z) + Σ I_j,v(x, y, z, t) + C_v(x, y, z)
```

其中 `B_v` 為背景場，`I_j,v` 為第 `j` 個設備的影響函數，`C_v` 為由感測器殘差推估出的校正場。

本研究採用連續場概念建立模型，但在實作上將房間離散為 3D 採樣網格，以便進行數值計算、熱圖輸出與區域平均分析。

## 預期貢獻

本研究的預期貢獻包含：

1. 提出一個不依賴高成本 CFD 模擬的室內三因子空間數位孿生框架。
2. 建立可由少量角落感測器校正的空間場估計方法。
3. 將冷氣、窗戶與照明轉換為可比較的設備影響函數。
4. 提供目標區域導向的設備動作排序方法。
5. 產出可重現的 Python 原型與模擬案例，支援後續實體部署或論文延伸。

## 初步研究假設

1. 單一房間內的環境因素可用簡化連續場近似表示。
2. 冷氣、窗戶與照明的空間影響可用距離衰減與方向性權重近似。
3. 8 顆角落感測器足以對整體場偏移與一階空間梯度進行校正。
4. 溫度與照度是本研究的主要變數，濕度作為次核心變數保留。
5. 模型輸出的設備排序應具備可解釋性，而不僅是數值最佳化結果。

## 預期成果

研究完成後應包含以下成果：

1. 數位孿生模型公式與系統架構。
2. Python 模擬原型。
3. 8 組設備組合情境模擬。
4. 溫度、濕度與照度空間熱圖。
5. 校正前後誤差分析。
6. 候選設備動作排序結果。
7. 模型限制與未來實體部署建議。
