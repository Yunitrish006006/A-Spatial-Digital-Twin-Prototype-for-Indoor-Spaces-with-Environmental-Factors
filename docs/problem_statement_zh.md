# 研究問題定義

## 核心問題

本研究要解決的核心問題是：

**在室內空間中，許多會影響環境的家電或環境裝置並沒有連網能力，系統無法直接知道它們的狀態、出力或對空間的影響。因此，需要透過有限感測器觀測到的環境變化，學習這些非連網裝置對溫度、濕度與照度造成的影響，並利用學習後的影響模型更準確地控制或推薦控制三個環境變數。**

## 問題背景

智慧建築或智慧家庭系統通常假設設備可以被連線、讀取或控制，例如透過 Wi-Fi、Bluetooth、Matter、BACnet、Modbus 或廠商 API 取得設備狀態。然而在真實場域中，許多裝置並不具備這些能力。

例如：

- 傳統冷氣可能無法回報目前實際出風量或冷卻效果。
- 手動窗戶無法回報開啟程度。
- 一般燈具可能只能被牆壁開關控制，無法提供亮度回饋。
- 新增家具、遮光物或熱源也可能改變空間環境，但不會主動提供資料。

這些裝置雖然不連網，卻會持續影響空間中的溫度、濕度與照度。如果系統只依賴裝置 API，就無法建立完整的環境理解。

## 研究缺口

現有方法常見兩種假設：

1. 假設裝置是智慧裝置，可直接取得狀態或控制參數。
2. 假設有大量感測器或高精度模擬工具，可完整觀測空間狀態。

本研究關注的是更受限但更接近一般場域的情況：

- 裝置本身可能沒有連網能力。
- 感測器數量有限。
- 不使用高成本 CFD 或完整 BIM/BMS 系統。
- 仍希望能推估裝置對空間不同區域的影響。

## 研究目標

本研究目標可分為四層：

1. **感測層**：使用固定 8 顆角落感測器取得溫度、濕度與照度資料。
2. **學習層**：從裝置啟用前後的環境變化中估計裝置影響函數。
3. **數位孿生層**：建立單房間三因子空間場模型，估計任意位置的環境狀態。
4. **控制輔助層**：根據學習後的影響模型，對候選控制動作進行排序。

## 可形成的研究問題

### RQ1

在只有 8 顆角落感測器的條件下，是否能建立單房間溫度、濕度與照度的空間估計模型？

### RQ2

在家電或環境裝置沒有連網狀態回報的情況下，是否能從環境感測資料學習其對空間不同區域的影響？

### RQ3

學習後的裝置影響模型是否能提升對三個環境變數的控制決策，例如選擇開冷氣、開窗或開燈？

### RQ4

將數位孿生模型封裝為 MCP tools 後，是否能讓 AI client 以標準化方式查詢、模擬與使用控制推薦能力？

## 系統輸入與輸出

### 輸入

- 房間尺寸與目標區域。
- 8 顆角落感測器的溫度、濕度與照度資料。
- 裝置可能位置與類型，例如冷氣、窗戶、照明。
- 外部環境條件，例如室外溫度、濕度與日照。
- 候選控制動作，例如開冷氣、開窗、開燈。

### 輸出

- 任意座標的溫度、濕度與照度估計值。
- 各區域的三因子平均值。
- 裝置對空間場的估計影響。
- 候選控制動作排序。
- MCP tool 回應，供 AI client 使用。

## 與一般智慧家庭系統的差異

一般智慧家庭系統偏向「控制已知裝置」：

```text
Device API → device state → control command
```

本研究偏向「從環境反推裝置影響」：

```text
Sensor observations → environmental change → appliance impact learning → control recommendation
```

因此，即使裝置不是智慧裝置，只要它對環境造成可觀測變化，模型仍可逐步估計它的影響。

## 一句話版本

本研究建立一個 MCP-enabled 單房間數位孿生原型，透過有限角落感測器學習非連網家電對溫度、濕度與照度的影響，並利用該影響模型支援更準確的環境控制動作推薦。
